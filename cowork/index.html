Cowork vs Chat 的三大差異

📋
「新任務」取代「新聊天」
按鈕從 New Chat 變成 New Task — 你不是來閒聊的，你是來完成一件待辦事項的
📁
「資料夾」為工作空間
不用逐一上傳檔案，直接指定資料夾，AI 能自由瀏覽、讀取、新建、編輯其中所有檔案
🧠
主動追問 + 計畫模式
AI 不再像「過度熱切的實習生」埋頭猛衝，而是先提問、確認假設、再行動
Architecture

Cowork 三大元件

Skills + Connectors + Plugins = 你的 AI 工作流引擎

01
⚡

Skills

可被關鍵句觸發的獨立 Prompt。在 Cowork 中輸入「/」或「+」即可調用。不用記住放在哪個 Project — 說出需求就觸發。

02
🔌

Connectors

外部服務連接器（Google Drive、Gmail、Slack、Notion 等）。就是 MCP 在 Cowork 中的呈現方式 — 讓 AI 能操作你的工具。

03
📦

Plugins

將 Skills + Connectors + Sub-agents 打包成一個「套件」。安裝即可用，不需逐一設定。像 App 一樣一鍵安裝。

Skill = 你在 B 班學的那個 Skill ｜ Connector ≈ MCP 一鍵版 ｜ Plugin = 打包好的全套工具箱
Setup Guide

5 分鐘設定 Cowork

打開 Claude Desktop → 切到 Cowork Tab → 開始使用

1

打開 Cowork 分頁

開啟 Claude Desktop，上方會看到三個分頁：Chat / Cowork / Code。點擊「Cowork」。

Claude Desktop 上方三個 Tab：Chat ｜ Cowork ｜ Code
2

選擇工作資料夾

點擊左側「Select folder」指定你的工作資料夾。Cowork 會掃描資料夾內所有檔案作為上下文。建議先用一個小資料夾測試。

3

安裝你的第一個 Plugin

左側 → Customize → Browse Plugins → 選一個適合的 Plugin → Install。推薦先裝「Productivity」。

4

連接外部服務（Connectors）

在 Plugin 設定中連接你常用的服務：Google Drive、Notion、Slack 等。步驟與 MCP Level A 一鍵授權相同。

5

開始第一個 Task

點擊「New Task」，描述你想完成的事。Cowork 會主動追問細節，然後開始執行。

💡 提示：用「/」叫出已安裝的 Skills，用自然語言描述任務也可以

Hands-On

實作練習：三個上手場景

安裝完 Plugin → 實際操作 → 體驗 AI 協作的差異

場景一：用 Productivity Plugin 管理一週工作

開啟 Cowork → 指定工作資料夾 → New Task → 週計畫文件
試試這個 prompt：
「幫我整理這個資料夾裡所有的待辦事項和會議筆記，產出一份下週的工作計畫。按優先級排序，標出需要準備的資料。」
✨ Cowork 會自動：掃描資料夾 → 識別待辦 → 追問優先級 → 產出 Word/Markdown 文件

場景二：用 Marketing Plugin 批量產出社群內容

安裝 Marketing → 連接 Notion → /Brand Voice → 5 篇貼文草稿
試試這個 prompt：
「根據我資料夾裡的品牌指南和最近 3 篇部落格文章，幫我產出 5 篇 Threads 貼文草稿。語氣要像跟朋友聊天，每篇 200 字以內。」
✨ Cowork 會自動：讀品牌指南 → 分析文章風格 → 追問目標受眾 → 批量產出 → 存成檔案

場景三：用 Plugin Create 打造你的第一個自訂 Plugin

安裝 Plugin Create → 描述你的流程 → Claude 自動產出 → 你的專屬 Plugin
試試這個 prompt：
「我每次寫完部落格文章，需要做 4 件事：取標題、寫 SEO 摘要、產生 URL slug、產生配圖提示詞。幫我建一個 Plugin 自動化這個流程。」
✨ Plugin Create 會引導你：定義步驟 → 選擇 Connectors → 建立 Orchestrator → 測試 → 完成
Build

Plugin 的四個組成部分

理解結構，才能客製化自己的工作流

📦 my-blog-publisher/
├── README.md ← 說明書（像 GitHub 的 README）
├── 📂 skills/ ← 各個 Skill 獨立檔案
│ ├── title-generator.md
│ ├── seo-summary.md
│ ├── url-slug.md
│ └── image-prompt.md
├── orchestrator.md ← 指揮檔（定義執行順序）
└── plugin.json ← 中繼資料（名稱、版本、描述）
💡 關鍵概念：Orchestrator
Orchestrator 是 Plugin 的「大腦」。它定義：
・每個步驟的執行順序
・步驟之間的資料傳遞
・是否需要使用者確認才繼續下一步
・類似 Chain of Thought — 一步一步執行，確認後再進下一步
⚠️ 注意：Plugin 目前只能在 Cowork Tab 觸發
Skills 可以在一般 Chat 和 Cowork 中使用，但 Plugin（多 Skill 串聯）只能在 Cowork 中執行。

開啟 Cowork → 指定部落格資料夾 Cowork

🎯 實戰範例：AI 自動化週報

每週五下午用一句話觸發完整的週報產出流程

「幫我做本週的工作總結」 Cowork

Cowork 主動追問：「需要涵蓋哪些專案？有沒有特別想強調的成果？」

讀取 Google Calendar 本週行程 MCP

自動彙整本週所有會議、工作時段

讀取 Notion 專案進度 MCP

掃描各專案看板，抓取本週更新的任務和狀態變更

觸發週報撰寫 Skill Skill

按公司模板格式整理：完成事項 / 進行中 / 下週重點 / 風險項目

透過 Slack 分享週報 MCP

自動發送到 #weekly-report 頻道，@主管

心智模型
Skill 教 AI「怎麼思考這個任務」
MCP 讓 AI「能碰到你的工具」
Cowork 給 AI「一張桌子坐下來工作」

三者缺一不可。沒有 Skill，AI 只會通用回答；
沒有 MCP，AI 碰不到你的資料；
沒有 Cowork，AI 做完的東西帶不走。
